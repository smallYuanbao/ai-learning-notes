## 一、Sparse中的核心BM25 核心原理

**一句话总结**：BM25 是一种计算查询和文档关键词匹配程度的相关度打分算法。把查询分词后，计算每个词与文档的相关度，再累加得到最终分数。

**稀疏索引的工作流程**：分词（jieba）-》统计词频-》倒排记录-》权重计算（BM25）。

**输出的内容**：简单来说类似一个map，key是单次，value是出现的次数，可能有多个value，根据这个出现的次数来确定，这个单词是否稀有。

**和 BIM 的区别**：BIM（二元假设模型）只看词是否在文档中出现过（0/1），BM25 在此基础上引入了词的权重（IDF）、词频饱和、文档长度惩罚等因素，效果远优于 BIM。

**三个核心因素**：

1. **IDF（词的权重）**：
   - 公式：`IDF = log((N - n(qi) + 0.5) / (n(qi) + 0.5))`
   - N = 文档总数，n(qi) = 包含这个词的文档数
   - 词越稀有，n(qi) 越小，IDF 越大，权重越高
   - 举例：“对乙酰氨基酚”只在少数文档出现，权重高；“的”、“是”几乎每篇都有，权重趋近于 0

2. **TF（词频）**：
   - 词在文档中出现次数越多，相关度越高
   - 但有**饱和机制**：调节因子 k1（默认=1）控制词频增长的上限，不会无限加分

3. **文档长度惩罚**：
   - 公式：`K = k1 * (1 - b + b * (dl / avg(dl)))`
   - dl = 当前文档长度，avg(dl) = 平均文档长度，b = 调节因子（默认=0.75）
   - 文档越长，K 越大，相关度 R(qi,d) 就越低
   - 举例：同样出现“发烧”3 次，500 字的文档比 10000 字的文档得分更高

**和 TF-IDF 的区别**：TF-IDF 词频无上限、无文档长度惩罚；BM25 有词频饱和和文档长度归一化，更贴近真实检索场景。

**和我的项目关系**：
- Sparse 检索的数学本质就是 BM25 的变体。
- bge-m3 生成 Dense 向量时免费附带 Sparse token weights，一个模型就能同时做语义检索和关键词检索，不需要额外部署 BM25 引擎。


### 词的权重（IDF）和词频（TF）的关系

- **TF（词频）**：这个词在这篇文档里出现了几次（单篇内部统计）
- **IDF（权重）**：这个词在整个知识库里有多稀有（全库统计）
- **两者关系**：两者相乘决定最终贡献。高频常见词（如“宝宝”）TF 高但 IDF 低，权重被压低；低频稀有词（如“对乙酰氨基酚”）TF 低但 IDF 极高，最终贡献可能更大
- **类比**：实习生发言 10 次（TF 高）不如 CEO 发言 1 次（IDF 极高）

**如何计算每个词与文档的相关度**：IDF(当前词) * TF(当前词)。

### 什么是倒排索引

简单来说，直接通过某个词就能知道它大概在哪些区域。例子：比如书本末尾的单词索引表，我们查”大模型“，会直接知道它在哪几页出现。

正向索引：书本的目录，通过目录我们知道这个目录中都包含里哪些内容。

#### 为什么要用它来”粗筛“
在 RAG 中，主流的做法是将文本转为向量进行相似度计算，但是它存在以下缺点：
- 计算非常慢（消耗算力）：如果你有 100 万个文本块，用户提问时，系统必须用复杂的数学公式（如余弦相似度）把用户的问题和这 100 万个向量挨个计算一遍，在高并发或者大数据量时非常耗时。

- 对专有名词不敏感：如果用户搜一个特定的产品型号“iPhone 17 Pro Max”或人名“张三”，向量模型可能会因为“理解语义”，反而召回了一堆包含“苹果手机”、“李四”的无关文档，把最精准的原件漏掉了。


## 二、 Dense 最主流的优化算法 HNSW

HNSW 是目前稠密向量（Dense Vector）检索领域最核心、最主流的“近似最近邻搜索”（ANN, Approximate Nearest Neighbor）优化算法

### 它优化了什么

在没有 HNSW 之前，计算两个稠密向量（比如 BGE-M3 吐出的 1024 维数组）的相似度，必须把它们的所有维度进行点积运算。
- 如果不优化（Flat 暴力搜索）： 查找最近的文档需要将全库 N 个向量挨个算一遍，时间复杂度是 O(N)。当 N 达到百万级别，检索一次需要几秒甚至几分钟，RAG 应用会直接卡死。
- HNSW 的优化： 它利用“分层跳表图”的几何结构，把检索变成了“在地图上导航导航”。它不需要遍历全库，只需要顺着图连线计算几十次，就能找到最近的向量，时间复杂度直接降到了 \(O(\log N)\)。
- 结果： 将百万级数据的检索时间从几秒钟直接缩短到几毫秒。


### 稠密向量检索的三大优化算法流派
1. 基于图的算法，最核心的就是 **HNSW** 

2. 基于聚类的算法(IVF)，以IVE-FLAT、IVE-PQ为代表。

3. 基于树的算法，以Annoy为代表。

## 二、Dense/Sparse 融合策略

HNSW（Dense 的优化算法） 是目前向量数据库中最核心、最主流、速度最快的向量索引算法之一。


混合检索中，Dense（语义向量）和 Sparse（关键词权重）各自返回一个排序列表，融合策略决定如何将两者合并成最终排名。

### 策略对比总览

| 维度 | 线性加权 | RRF（倒数排名融合） |
|------|---------|-------------------|
| **核心思想** | 给两个分数各一个权重，加权求和 | 不看分数大小，只看排名位置 |
| **公式** | `最终分数 = α × Dense分数 + (1-α) × Sparse分数` | `RRF = Σ 1/(60 + rank_i)` |
| **是否需要归一化** | 需要（Dense 0~1，Sparse 无上限，量纲不同） | 不需要（只关心排名，不关心分数值） |
| **是否需要调参** | 需要调 α（语义 vs 关键词的权重） | 不需要 |
| **优点** | 简单直观，可灵活控制语义和关键词的相对重要性 | 无需调参，天然解决量纲不一致问题 |
| **缺点** | α 选值靠经验；两套分数必须归一化后才能相加 | 丢失了具体分数信息（只保留排序，不保留差距大小） |
| **适用场景** | 你能明确说清“语义更重要”还是“关键词更重要”时 | 不确定两者权重，或两套分数量纲差异太大时 |


### 策略一：线性加权

**核心思想**：给 Dense 和 Sparse 各一个权重，加权求和。α 越接近 1，越偏重语义；α 越接近 0，越偏重关键词。

**公式**：
最终分数 = α × Dense分数 + (1-α) × Sparse分数

**举例**（α=0.7）：
- 文档 A：Dense=0.9，Sparse=0.3 → 混合 = 0.7×0.9 + 0.3×0.3 = 0.63 + 0.09 = **0.72**
- 文档 B：Dense=0.4，Sparse=0.9 → 混合 = 0.7×0.4 + 0.3×0.9 = 0.28 + 0.27 = **0.55**

文档 A 胜出，因为 Dense 权重（0.7）高于 Sparse（0.3）。

### 归一化（Min-Max）

**为什么需要**：Dense 余弦相似度（0~1）和 Sparse BM25 分数（0~无穷大）量纲不同，不归一化会导致 BM25 分数碾压余弦相似度，Dense 完全失效。

### 为什么必须归一化：一个具体例子

**场景**：用户问“宝宝发烧，对乙酰氨基酚怎么吃”，知识库有两篇文档：

| 文档 | Dense 分数（0~1） | Sparse 分数（无上限） |
|------|:---:|:---:|
| 文档 A：《小儿发热护理指南》 | **0.92**（语义高度匹配） | 3.5（部分关键词命中） |
| 文档 B：《对乙酰氨基酚剂量表》 | 0.65（语义中等匹配） | **28.7**（关键词完美命中） |

**不归一化，直接加权（α=0.7）**：
- 文档 A = 0.7×0.92 + 0.3×3.5 = 0.644 + 1.05 = **1.694**
- 文档 B = 0.7×0.65 + 0.3×28.7 = 0.455 + 8.61 = **9.065**

❌ **问题**：文档 B 的 Sparse 分数（28.7）巨大，直接碾压了 Dense 信号。Dense 最相关的文档 A 反而排到第二，α=0.7 形同虚设。

**先归一化，再加权（α=0.7）**：
- Sparse 分数归一到 0~1：A→0.122，B→1.000
- 文档 A = 0.7×0.92 + 0.3×0.122 = 0.644 + 0.037 = **0.681** ✅ 排名第一
- 文档 B = 0.7×0.65 + 0.3×1.000 = 0.455 + 0.300 = **0.755** 排名第二

✅ **结果**：Dense 语义最相关的文档 A 排到第一，Sparse 精确匹配的文档 B 紧接其后。α 真正发挥了控制权重的作用。

**一句话总结**：归一化把不同量纲的分数拉到同一起跑线，让你的权重参数真正说了算。

**公式**：`归一化值 = (原始值 - 最小值) / (最大值 - 最小值)`

**效果**：所有分数映射到 0~1 之间，和余弦相似度处于同一量级，α 能真正发挥作用。

**注意事项**：
- Dense 的余弦相似度（0~1）和 Sparse 的 BM25 分数（无上限）量纲不同，必须先归一化到同一范围（如 Min-Max 映射到 0~1）后再加权
- α 的具体值需要通过小规模实验来确定——拿 5-10 个问题，用不同 α 值跑一遍，看哪个检索命中率最高

**适用场景**：
- 你能明确说出语义更重要还是关键词更重要时
- 需要灵活调节两者权重的场景（比如医疗场景下，药品名需要精确匹配，α 可以调低；育儿日常问答更依赖语义泛化，α 可以调高）


### 其他归一化方法简介

除了 Min-Max，还有两种常见的归一化方法：

| 方法 | 公式 | 适用场景 | 你的项目用吗 |
|------|------|---------|:---:|
| **Min-Max 归一化** | `(x - min) / (max - min)` | 数据分布不规则、需要固定范围 0~1 | ✅ 使用 |
| **Z-score 标准化** | `(x - 均值) / 标准差` | 数据服从正态分布 | ❌ 不适用 |
| **Sigmoid 归一化** | `1 / (1 + e^(-x))` | 需要平滑压缩、不在意绝对大小关系 | ❌ 不需要 |

**为什么选 Min-Max**：
- Sparse 的 BM25 分数通常是长尾分布——少数文档分数很高，大部分集中在低分段。Min-Max 不受分布形态影响，直接映射到 0~1。
- Z-score 要求数据近似正态分布，BM25 分数不满足这个条件。
- Sigmoid 会把所有值压缩到 0~1，但会扭曲原始分数的相对大小关系，而线性加权需要保留这个相对关系。

**一句话记忆**：
> 数据范围固定、分布不规则 → Min-Max；数据正态分布 → Z-score；需要平滑非线性压缩 → Sigmoid。

### 策略二：RRF（倒数排名融合，Reciprocal Rank Fusion）

**核心思想**：不关心 Dense 和 Sparse 各自给了多少分，只关心每个文档在各自列表中的排名位置。两个列表里都排得靠前的文档，最终得分更高。

**为什么需要 RRF**：线性加权的最大问题是 Dense 和 Sparse 分数量纲不同（一个 0~1，一个无上限），直接相加会扭曲结果。RRF 只关心排名，天然绕过了这个问题。

**公式**：

RRF分数 = 1/(60 + Dense排名) + 1/(60 + Sparse排名)

- `60` 是平滑因子，防止排名第一的文档得分过高，让整个融合更稳定

**举例**：
- 文档 A：Dense 排名第 1，Sparse 排名第 5
  - RRF = 1/(60+1) + 1/(60+5) = 0.0164 + 0.0154 = **0.0318**
- 文档 B：Dense 排名第 3，Sparse 排名第 1
  - RRF = 1/(60+3) + 1/(60+1) = 0.0159 + 0.0164 = **0.0323**

文档 B 略胜一筹，因为它在两个列表中都排在前列，说明它是“既语义相关又关键词匹配”的文档。

**适用场景**：
- 你不确定语义和关键词哪个更重要时
- 两套分数量纲差异太大，不想做归一化时
- 需要更鲁棒的融合，不希望某一路的极端高分扭曲最终结果
