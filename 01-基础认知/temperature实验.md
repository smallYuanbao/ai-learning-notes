# Temperature 与 Top-P 实验

> **日期**：2026-07-12
> **来源**：自己的实验（baby-ai 项目）
> **关联面试题**：提示词工程专题 第 9 题「temperature 与 top_p 参数的作用及选型方法」

## 一句话总结
Temperature 控制概率分布的平坦程度（调“地形起伏”），Top-P 控制候选 token 的累积概率截止线（画“截止线”）。两者不改变词表大小，只改变“选谁的概率大”。

## 实验一：固定 temperature=0.7，对比 top_p

**问题**：宝宝发烧 38.5 度需要吃药吗？

| 参数 | 回答特征 |
|------|---------|
| top_p=0.1 | 保守，用词集中在高频表达，结构是线性叙述 |
| top_p=0.9 | 更丰富，自动生成了表格和颜色分级，语气也更有层次 |

**发现**：top_p 越低候选词越少、输出越保守，但不影响概率分布形状。

## 实验二：固定 top_p=0.9，对比 temperature

**问题**：同上

| 参数 | 回答特征 |
|------|---------|
| temperature=0.0 | 高度确定、可复现，适合需要准确性的场景 |
| temperature=0.7 | 兼顾可靠性和表达丰富度，当前项目使用 |
| temperature=1.5 | 出现幻觉式引用（编造“参考资料 2 和 3”），不适合生产环境 |

**发现**：temperature 太高会拉平概率分布，让低概率的不靠谱词混进候选池，即使 top_p 不高也可能输出幻觉。

## 核心理解

LLM 生成下一个 token 时，面对的是一个固定不变的词表（约 10 万个 token）。
- **temperature** 不改变集合，只改变概率分布的形状：低则尖锐（高质量 token 独占），高则平坦（更多 token 分摊）
- **top_p** 在调整后的分布上画截止线：从概率最高的 token 开始累加，累到累积概率达到 p 就停
- **两者配合**：temperature 先决定分布形态，top_p 再圈定候选范围。如果 temperature 太高把分布拉平了，即使 top_p 设得很低，候选池里也可能混进原本不该出现的词

## 面试怎么讲

“我在项目里做了两组对照实验。固定 temperature 对比 top_p，发现 top_p 越低输出越保守。固定 top_p 对比 temperature，发现 0.7 平衡最佳，1.5 出现了幻觉。最终选型 temperature=0.7 + top_p=0.9。我的理解是：整个词表固定不变，temperature 调概率分布形状，top_p 画截止线，两者配合控制输出的随机性和质量。”
